# 🚀 البدء السريع - Supabase Version

## ⚡ 15 دقيقة لتشغيل النظام

---

## الخطوة 1️⃣: التحقق من الاتصال (1 دقيقة)

```bash
npm run verify:supabase
```

يجب أن ترى:
```
✅ NEXT_PUBLIC_SUPABASE_URL
✅ NEXT_PUBLIC_SUPABASE_ANON_KEY
✅ SUPABASE_SERVICE_ROLE_KEY
```

إذا كان هناك ❌ جنب أي متغير = توقف! ادخل Settings > Vars وتأكد منهم

---

## الخطوة 2️⃣: إنشاء الجداول في Supabase (5 دقائق)

### الطريقة الأسهل:

1. افتح: https://app.supabase.com
2. اختر مشروعك
3. انقر **SQL Editor** (الشريط الجانبي)
4. انقر **New Query**
5. افتح ملف: `scripts/001_create_products_table.sql` من مشروعك
6. انسخ **كل المحتوى**
7. ألصقه في SQL Editor
8. انقر **Run** (زر أزرق)
9. انتظر رسالة النجاح ✅

---

## الخطوة 3️⃣: تشغيل المشروع (1 دقيقة)

```bash
npm run dev
```

افتح http://localhost:3000

---

## الخطوة 4️⃣: تسجيل الدخول إلى لوحة التحكم (2 دقيقة)

```
1. اذهب إلى: http://localhost:3000/admin/dashboard
2. البريد: admin@example.com
3. كلمة المرور: password123
4. اضغط: "دخول"
```

---

## الخطوة 5️⃣: إضافة أول منتج (3 دقائق)

```
1. في لوحة التحكم، انقر: "+ إضافة منتج"
2. ملء الحقول:

   📝 اسم المنتج
   مثال: كاميرا Canon EOS R5
   
   📄 الوصف
   مثال: كاميرا احترافية عالية الدقة
   
   🏷️ النوع
   اختر: sale (بيع) أو rent (إيجار)
   
   📍 الموقع
   اختر: sale أو rent أو equipment
   
   💰 السعر
   مثال: 3500
   
   🖼️ الصورة
   حمّل صورة

3. انقر: "إضافة المنتج"
```

✅ المنتج الآن محفوظ في Supabase!

## 🚨 ماذا لو حدثت مشاكل؟

### المشكلة: "Failed to fetch products"

```
❌ السبب: الجداول لم تُنشأ بعد
✅ الحل: 
   1. افتح Supabase Dashboard
   2. SQL Editor > New Query
   3. شغّل: scripts/001_create_products_table.sql
```

### المشكلة: "Invalid API Key"

```
❌ السبب: متغيرات البيئة غير موجودة
✅ الحل:
   1. Project Settings > Variables
   2. تأكد من وجود جميع المتغيرات
   3. أعد تحميل المتصفح (Ctrl+Shift+R)
```

### المشكلة: "Relation does not exist"

```
❌ السبب: لم تشغل SQL بعد
✅ الحل: كرر خطوة إنشاء الجداول
```
